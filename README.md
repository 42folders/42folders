I build and ship software to real users. My current project is a desktop application that runs daily in a bicycle repair shop — handling inventory, repair jobs, invoicing, GST accounting, and monthly P&L for a live business.

While I am well-versed in ETL, data wrangling and reporting, building the bike shop system pushed me to make design and engineering decisions that come with production software — money precision, data integrity, cross-platform deployment, and real incident handling.

---

### Featured project

#### [Bike Shop Manager](https://github.com/42folders/bike-shop-manager-public)

A local desktop application for a real bicycle repair shop. It replaced pen and paper with a single, trustworthy system to track daily operations.

**What it covers:** inventory with bin locations and low-stock alerts · repair jobs with deposits and task checklists · customer-facing PDF invoices and receipts · supplier purchasing with PDF invoice import and fuzzy part-matching · GST-inclusive and GST-exclusive invoice handling · monthly expenses and P&L · cash vs bank reconciliation · exportable reports

`Python 3.12` · `PySide6` · `SQLAlchemy 2.0` · `SQLite` · `Alembic` · `pdfplumber` · `PyInstaller` · `pytest` 

**[Read the case study →](https://github.com/42folders/bike-shop-manager-public/blob/main/docs/CASE_STUDY.md)** — the problem, constraints, architecture decisions, and what I learned operating it in production.
